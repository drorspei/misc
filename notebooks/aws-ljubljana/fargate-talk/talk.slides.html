Wait what's AWS Fargate?

  • Serverless docker containers
  • No server setup or management required
  • Auto-handles scaling and infrastructure

Ideas for faster startup?

Build:

  • Remove unnecessary files, e.g. caches, node_modules, use multistage builds
  • Use smaller base, e.g. alpine instead of ubuntu
  • Organize layers to have more cache hits

Post build:

  • deduplicate files - turn copies into links
  • use zst instead of gzip
  • reorder files in layers' tars
  • redistribute files across layers

Compute optimal layer sizes using linear algebra

In[29]:
D = E = 225
A = [
    [1,                  1,    1], # sum to 2700
    [1/E+1/(D/2), -1/(D/2),    0], # layer 1 finishes extracting = layer 2 finishes downloading
    [0,            1/E+1/D, -1/D]  # layer 2 finishes extracting = layer 3 finishes downloading
]
x = [2700, 0, 0]
np.linalg.solve(A, x)
